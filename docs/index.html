🖼️ Flux Wallpaper Generation Pipeline

A self-hosted text-to-image pipeline that wraps HuggingFace Diffusers around Black Forest Labs' FLUX models to generate high-resolution desktop wallpapers — running on Apple Silicon (MPS) and auto-upgrading to CUDA, with memory-aware GGUF quantization, LoRA support, and a daily off-hours scheduler.

FLUX.2-klein · FLUX.1-schnell Diffusers + GGUF MPS / CUDA / CPU auto Real-ESRGAN upscale LoRA-ready No telemetry
View on GitHub →

What it does

Two model tiers

Fast dev tier (FLUX.2-klein-4B, Q4 GGUF) for iteration; quality prod tier (FLUX.1-schnell) for the daily wallpaper — one Diffusers code path, swap via config.

Runs anywhere

Auto-detects cuda → mps → cpu, picks bf16/fp32, and selects the GGUF quant (Q8/Q6/Q5/Q4) that fits your memory. CPU-offloads heavy tiers when RAM is tight.

Wallpaper-shaped

Generates at a FLUX-native size then upscales (Real-ESRGAN → Lanczos fallback) to 1080p / 1440p / 4K. Every image gets a reproducible JSON sidecar.

LoRA + scheduler

Stack third-party / local LoRAs or train your own. A launchd/cron job makes one wallpaper a day in off-hours and sets your desktop.

Design decisions

GGUF, not FP8/NF4

GGUF is the only quantization that runs on both MPS and CUDA. FP8 (torchao) and NF4 (bitsandbytes) are CUDA-only, so they're deliberately unused — one quant path everywhere.

guidance_scale = 0

FLUX schnell/klein are guidance-distilled — they generate in ~4 steps with no CFG. Non-zero guidance degrades them. The defaults reflect that.

LoRA without fusing

Fusing LoRA into GGUF-quantized weights is unreliable, so requesting any LoRA transparently loads an unquantized base and applies adapters via PEFT scaling.

CPU torch by default

torch lives in conflicting cpu/cuda dependency-groups — the multi-GB CUDA stack is never installed unless asked for, keeping Docker images small.

Project layout

fluxwall/
  device.py     cuda>mps>cpu · dtype · quant auto-select
  models.py     unified loader (FLUX.2 / FLUX.1 GGUF / SDXL) + fallback
  loras.py      LoRA stack resolution + scaling (GGUF-aware)
  config.py     profiles · models · resolutions · lora stacks
  prompts.py    prompt files + deterministic daily rotation
  pipeline.py   generate → upscale → organise
  organize.py   output/<date>/slug.png + JSON sidecar
  upscale.py    Real-ESRGAN → Lanczos fallback
  webui.py      Gradio UI (telemetry off, localhost)
  cli.py        generate · batch · daily · loras · train · ui
  scheduler/    daily_wallpaper.py + launchd/cron templates
  train/        captioned-image dataset → ak:ml-finetune

Quickstart

uv sync --extra ui --extra upscale --extra lora     # Mac / dev (CPU/MPS torch)

uv run python -m fluxwall generate "northern lights over a fjord" -r uhd
uv run python -m fluxwall batch prompts/wallpapers.txt
uv run python -m fluxwall ui                          # localhost Gradio UI
uv run python -m fluxwall generate "neon city" --lora some-org/neon:0.8
